Basically, JavaScript executes in a execution context which has 2 part. One is, variable environment, another is thread of execution. Variable part is also called memory part where thread execution part is called coding part. JS creates a new execution context when JavaScript code runs. Memory part take names from the code and make variable undefined. After the mechanism of memory part, it goes to execution context part. This part gives the value of memory and all the method along with calculation we want.

The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks.

An example of this is the setTimeout method. When a setTimeout operation is processed in the stack, it is sent to the corresponding API which waits till the specified time to send this operation back in for processing. Where does it send the operation? The event queue.

Most of the things are similar about local storage and session storage. But there are some differences between them.

Local Storage:

  • Can store 10 MB data.
  • Stores data with no expiration date
  • Lasts until the user deletes it.

Session Storage:

  • Can store 5 MB data.
  • Destroyed once the user closes the browser
  • Only accessible while the window that have created it is open
